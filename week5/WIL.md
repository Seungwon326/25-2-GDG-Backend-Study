# 5주차 - SOLID 원칙과 스프링 빈

- 객체 지향 프로그래밍 - 컴퓨터 프로그램을 객체들의 상호작용으로 파악하고자 하는 컴퓨터 프로그래밍의 패러다임
    - 유연하고 변경에 용이
    - 특징: 추상화, 캡슐화, 상속, 다형성
        - 다형성 - 어떤 역할에 대해 여러 가지 구현 방식이 존재
- SOLID 원칙
    - SRP - 단일 책임 원칙
        - 하나의 클래스는 단 한 개의 책임을 가져야 함
    - OCP - 개방-폐쇄 원칙
        - 클래스는 확장에 열려있어야하며, 수정에는 닫혀있어야 함(기존 코드는 변경하지 않고 기능을 확장)
    - LSP - 리스코프 치환 원칙
        - 자식 클래스는 언제나 부모 클래스를 대체할 수 있어야 함
    - ISP - 인터페이스 분리 원칙
        - 인터페이스를 각각 사용에 맞게 잘게 분리해야 함(하나의 인터페이스에는 하나의 기능만)
    - DIP - 의존관계 역전 원칙
        - 추상화에 의존하고, 구체화에 의존하면 안됨 → 각 클래스 간의 결합도를 낮추기 위함
    - SOLID 원칙을 지키기 위해 의존성 주입을 지원
- IOC - 객체 생성 및 관리에 대한 제어권을 프레임워크가 가져감
- DI - 인터페이스에만 의존, 구현 객체는 외부에서 주입
    - 느슨한 결합 → 클래스 간 영향력 최소화
- 의존성 주입 방법
    - 생성자 주입 → 권장 방법
        - 생성자 호출 시점에 1번만 주입
        - 객체의 주입을 강제
        - 불변성
    - 수정자 주입
        - 변경 가능성이 있어 의존 관계에 사용
    - 필드 주입
        - 의존성이 변경될 가능성이 있음
